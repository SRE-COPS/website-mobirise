Introduction

Git is a distributed version control system that enables multiple developers to work on a single project simultaneously without interfering with each other's work. GitHub, an online platform for hosting Git repositories, further enhances collaboration, allowing for code review, issue tracking, and more.

Registering on GitHub

To collaborate and share your projects with others, you'll first need to create a GitHub account:

  1. Visit GitHub's sign-up page.
  2. Fill out the form with your details and verify your email address.
  3. Once verified, you can start using GitHub to host your repositories.

Generating a New Token in GitHub

A Personal Access Token (PAT) is necessary for authenticating access to GitHub through the command line:

  1. Go to your GitHub settings.
  2. Click on "Developer settings" and navigate to "Personal access tokens".
  3. Click "Generate new token", select the necessary scopes, and generate it.
  4. Copy and securely store the token; you won't be able to see it again!

Note: Treat your tokens like passwords and keep them secret.

Adding SSH Keys

SSH keys provide a secure way of connecting to GitHub without needing to enter your username and password every time:

  1. Open a terminal and generate a new SSH key with ssh-keygen -t ed25519 -C "your_email@example.com".
  2. Follow the prompts to select where to save the key and enter a passphrase.
  3. Add the SSH key to your GitHub account under "SSH and GPG keys" in your settings.

Installing Git

Before you can use Git, you'll need to install it on your machine:

  • For Windows, download the installer from git-scm.com.
  • On macOS, use Homebrew: brew install git.
  • For Linux, install Git using your package manager, e.g., sudo apt-get install git for Ubuntu/Debian.

Creating and Cloning a Repository

Repositories are fundamental to GitHub, serving as the central location where all your project's files, including its revision history, are stored. Follow these steps to create a new repository on GitHub and then clone it to your local machine.

Creating a Repository on GitHub

  1. Sign in to GitHub: Navigate to github.com and log in to your account.
  2. Create a New Repository: Click on the "+" icon in the top-right corner and select "New repository" from the dropdown menu.
  3. Configure Your Repository:
    • Name your repository (e.g., "my-first-repo").
    • Add a description (optional but recommended).
    • Choose to make the repository public or private.
    • Initialize with a README, .gitignore, or license if desired.
  4. Create Repository: Click the "Create repository" button.

Cloning a Repository to Your Local Machine

Cloning creates a local copy of your repository for development. Here's how:

  1. Open Terminal (or Git Bash on Windows): Navigate to the directory where you want your project to be located.
  2. Clone the Repository: Use the git clone command followed by the URL of your repository. git clone https://github.com/yourusername/my-first-repo.git
  3. Navigate into Your Repository: Change directory to your newly cloned repo to start working. cd my-first-repo

Note: You can find your repository's URL on its GitHub page, under the "Code" button.

Basic Commands

Understanding Git's basic commands is crucial for managing versions of your project efficiently. Here, we'll dive into essential commands with detailed steps and examples.

1. Creating a New File

Step: Use a text editor or terminal command to create a new file in your repository directory.

Example: Creating a simple HTML file named index.html.

echo "<!DOCTYPE html><html></html>" > index.html

Exercise: Create a new file named about.html with basic HTML structure.

2. Staging Changes with git add

Step: After creating or modifying files, use git add to stage them for commit.

Example: Staging the newly created index.html file.

git add index.html

Exercise: Stage the about.html file you created in the previous step.

3. Committing Changes with git commit

Step: Commit your staged changes to the repository, including a meaningful commit message describing what was changed.

Example: Committing the staged index.html with a message.

git commit -m "Add initial index.html file"

Exercise: Commit the staged about.html file with an appropriate commit message.

4. Pushing Changes with git push

Step: After committing your changes, use git push to upload them to a remote repository (like GitHub).

Example: Pushing committed changes to the remote repository's main branch.

git push origin main

Exercise: Push your commit containing the about.html file to GitHub.

Note: Before pushing, ensure you have set up the remote repository and are authorized to push changes. If you're working with a newly cloned repository, the remote is usually set up for you.

Setting Up a Remote Repository and Pushing Changes

This section walks through creating a new repository on GitHub, cloning it, making changes, and pushing those changes back to the remote repository.

1. Creating a Remote Repository on GitHub

  • Sign in to GitHub and click "New" next to Repositories.
  • Fill in the repository name, description, and visibility settings.
  • Click "Create repository".

2. Cloning the Repository

  • Copy the repository URL from GitHub's "Code" button.
  • Open a terminal and run: git clone <repository-URL>

3. Making Changes and Committing Locally

  • Navigate into the cloned repository: cd <repository-name>
  • Create or edit files. Example for creating a README.md: echo "# My Awesome Project" > README.md
  • Stage the changes: git add README.md
  • Commit the changes: git commit -m "Initial commit with README"

4. Pushing Changes to GitHub

  • Ensure you're authorized to push (via SSH keys or HTTPS credentials).
  • Push the commit: git push origin main

Note: When cloning a repository, Git automatically sets up a remote named "origin." Ensure you have write access to push changes. For HTTPS, you may need your GitHub credentials or a personal access token.

Working with Branches

Branching in Git allows you to diverge from the main line of development and work on new features or fixes without affecting the stable version. This section provides a step-by-step guide to creating, switching, and merging branches, complemented by examples and exercises.

Creating a New Branch

  • Example: To create a branch named "feature-x", use the following command:
    • git branch feature-x
  • This command creates a new branch but keeps you in your current branch.

Switching Between Branches

  • Example: To switch to the "feature-x" branch, use the command:
    • git checkout feature-x
  • Now, you are in the "feature-x" branch, and any commits you make will be on this branch.

Merging Branches

  • After completing work on a branch, you can merge it back into the main branch (e.g., "main").
  • Example: To merge "feature-x" into "main", first ensure you are in the "main" branch, then merge:
    • git checkout main
        git merge feature-x
  • This merges the changes from "feature-x" into "main".

Exercise: Creating and Merging a Branch

  • Create a new branch named "new-feature" from "main".
  • Switch to the "new-feature" branch.
  • Create a new file named "feature.txt" and add some text to it.
  • Stage and commit the new file to "new-feature".
  • Switch back to the "main" branch.
  • Merge the "new-feature" branch into "main".

Solution to Exercise

  • Step 1: Create the branch with git branch new-feature.
  • Step 2: Switch to it using git checkout new-feature.
  • Step 3: Create "feature.txt", add content, and save the file.
  • Step 4: Stage with git add feature.txt, then commit with git commit -m "Add feature.txt".
  • Step 5: Switch back to "main" with git checkout main.
  • Step 6: Merge the branches with git merge new-feature.

Following these steps will merge the "new-feature" branch changes into "main", including the new "feature.txt" file.

Git Pull and Push

Understanding how to synchronize your local repository with a remote repository is crucial. This section covers `git pull` to fetch and merge changes from the remote repository and `git push` to upload local repository changes to a remote repository.

Using git pull

Example: Fetch and merge changes from the remote repository to your current branch.

git pull origin main

This command fetches changes from the 'main' branch of the remote named 'origin' and merges them into your current local branch.

Using git push

Example: Push your committed changes from your local repository to the remote repository.

git push origin main

Ensure you have committed any changes before pushing. This command uploads your commits to the 'main' branch of 'origin'.

Exercise: Synchronize Your Repository

  • Make a change in your local repository (e.g., add a file or update content).
  • Commit your change locally.
  • Use git pull to update your local repository with the latest changes from the remote repository.
  • Finally, push your changes back to the remote repository with git push.
